# Session: ghcacher-http-unification

## Goal
Unify ghcacher's two protocols (Unix socket + HTTP) into HTTP only, add subscription features, harden config requirements, and add operational controls.

## Current State
All work committed through multiple commits on main. 68/68 tests passing.

## Problem/Context
Previous session left a Unix domain socket (serve.rs) for change_log broadcast alongside the HTTP cmd server. User wanted one protocol, plus several feature additions.

## Solution/Approach

### Completed this session

1. **Unix socket → SSE** (`serve.rs` deleted):
   - `src/cmd.rs` absorbs broadcast loop: `GET /events` registers persistent `TcpStream` as `SseClient`
   - 500ms poll loop pushes `id: N\ndata: {...}\n\n` frames to all clients
   - `Last-Event-ID` header on reconnect sets `client.last_sent_id` -- server replays missed rows automatically
   - `Cmd::Serve` removed from CLI; HTTP server is always part of `ghcache watch`
   - `ghcache-client` gains `EventStream::new(port).subscribe(|ev| ...)` in `tail.rs`

2. **`--no-sync` flag** on `ghcache watch`:
   - Spawns HTTP server + SSE, parks main thread; sync loop disabled
   - Useful for read-only event consumers without needing full sync

3. **`/pause` and `/resume` endpoints**:
   - `Arc<AtomicBool>` shared between cmd server and sync loop
   - Watch loop checks flag at top of each iteration (500ms polling while paused)

4. **`notifications: bool` on subscriptions**:
   - `POST /subscribe` accepts `notifications: bool` alongside `pr_sync: bool`
   - `active_notifications_repos()` on `Subscriptions` (mirrors `active_pr_sync_repos`)
   - Fed into `notifications::sync()` each watch iteration as `extra_notif_slugs`

5. **`staging_folder` required**:
   - `validate()` bails with `"staging_folder is required in [global] config"` if absent
   - Type changed from `Option<PathBuf>` to `PathBuf` throughout `ResolvedConfig`
   - New config test: `fails_without_staging_folder`

6. **`ghcache init` writes to XDG config path**:
   - Now writes to `~/.config/ghcache/config.toml` (via `dirs::config_dir()`)
   - Falls back to `./ghcache.toml` if config_dir unavailable
   - Creates directory if needed

## Key Insights
- SSE over raw TcpStream is straightforward: write headers once, keep connection open, write `id:/data:/blank` frames
- `Last-Event-ID` on reconnect maps directly to `change_log.id` -- backfill is implicit in the per-client `last_sent_id` cursor
- `Arc<AtomicBool>` for pause is shared between cmd server thread and watch loop; no channel needed
- `staging_folder` being `PathBuf` (not Option) eliminates a class of unwrap/if-let throughout the codebase
- `dirs::config_dir()` returns `~/.config` on Linux, `~/Library/Application Support` on macOS
