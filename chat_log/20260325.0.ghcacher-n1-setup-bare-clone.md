# Session: ghcacher-n1-setup-bare-clone

## Goal
Performance and architecture improvements to ghcacher -- a local GitHub data cache backed by SQLite that syncs via the `gh` CLI.

## Current State
All work committed at d7bc6ca on main. 68/68 tests passing. Binary builds clean.

## Problem/Context
- All N+1 SQL query patterns eliminated across sync modules
- Go huh TUI replaced with native Rust inquire form
- Rate limit thresholds now configurable in [global] config
- Config auto-creates db directory instead of erroring
- Discussed but not yet implemented: bare clone architecture + command socket for on-demand repo provisioning

## Solution/Approach

### Completed this session
1. **N+1 elimination** across all sync modules:
   - `db.rs::upsert_repo`: execute + SELECT → RETURNING id
   - `events.rs`: SELECT after INSERT OR IGNORE → last_insert_rowid()
   - `branches.rs`: preload existing names HashSet + RETURNING id
   - `notifications.rs`: preload repo_id HashMap + existing gh_ids HashSet + RETURNING id + removed get_repo_id per row
   - `prs.rs`: preload PR numbers HashSet + RETURNING id in both sync() and sync_targeted()
   - `checkout.rs`: 2 query_rows per task → 1 batch JOIN across all tasks grouped by unique repo_id

2. **Rust TUI setup**: Replaced Go subprocess + TempBinary + JSON round-trip with native inquire form. build.rs is now empty.

3. **Configurable rate limits**: `rate_warn_threshold` and `rate_stop_threshold` in [global] config, stored on GhClient instead of hardcoded constants.

4. **Config auto-creates db directory**: create_dir_all instead of bail.

### Designed but not implemented
- **Bare clone architecture**: Replace per-branch working tree checkouts with one bare clone per repo at `{staging}/{owner}/{name}.git`. Consumer apps use git2 crate to read files from any branch in-process via mmap (zero-copy). No working tree needed.
- **Command socket** (`ghcache.cmd.sock`): Request/response Unix socket alongside existing event broadcast socket (`ghcache.sock`). Consumer sends `{"cmd":"watch","owner":"myorg","repo":"backend"}\n`, ghcacher bare-clones if needed, responds `{"status":"ready","path":"/path/to/repo.git"}\n`. Simple NDJSON framing, one connection per request.
- **ghcache-client**: Add `ensure_repo(owner, repo) -> Result<PathBuf>` that connects to cmd socket and returns bare clone path.

## Key Insights
- git bare clones store all branch objects on disk; `git2` crate mmaps pack files for zero-copy blob reads
- Consumer app never needs a working tree -- just the bare clone path + git2
- `staging_folder` in config is the shared contract between ghcacher and consumer apps
- Command socket is request/response (one conn per req, close after response) -- simpler than the broadcast event socket
- Unix SOCK_STREAM is ordered and reliable; NDJSON framing (newline-delimited) handles message boundaries
- The distinction between the two sockets: `ghcache.sock` = push events, `ghcache.cmd.sock` = pull/provision requests
