# Session: ghcacher-rusqlite-to-sqlx-migration

## Goal
Migrate the entire ghcacher workspace from rusqlite (sync) to sqlx 0.8 (async) + tokio 1 to eliminate the libsqlite3-sys version conflict between rusqlite 0.38 (libsqlite3-sys 0.36) and sqlx 0.8 (libsqlite3-sys 0.28/0.30) and achieve library parity with ghcache-client.

## Current State
COMPLETE. Committed as d2a07a9. 24 files changed, +3377/-1983 lines. 72 tests pass, 0 failures.

## Problem/Context
rusqlite 0.38 links libsqlite3-sys 0.36, sqlx 0.8 links libsqlite3-sys 0.30. These cannot coexist in one Cargo workspace due to a hard native link conflict. ghcache-client was already migrated to sqlx in a previous session; the main ghcache binary needed to follow.

## Solution/Approach
Full rewrite of all 15 src/ files plus ghcache-client:
- GitHubClient trait: #[async_trait] for dyn dispatch (native RPITIT can't do dyn)
- MockGhClient: RefCell<VecDeque> -> std::sync::Mutex (Send requirement for async)
- DB pattern: entry points take &SqlitePool, sub-functions take &mut SqliteConnection
- Per-repo transactions: pool.begin() -> &mut *tx coercion -> tx.commit()/rollback()
- conn.last_insert_rowid() -> SELECT last_insert_rowid() on same connection
- raw_bind_parameter for IN clauses -> sqlx::QueryBuilder::push_bind
- std::thread::spawn for checkout -> tokio::spawn + tokio::process::Command
- cmd.rs: std::net::TcpListener -> tokio::net with async IO, SSE via OwnedWriteHalf
- main.rs: fn main() + Runtime::new().block_on() (daemonize before tokio starts)
- query/sql.rs: ValueRef::type_info().name() + try_get cascade for generic columns
- open_in_memory(): max_connections(1) -- tests must drop(c) before &pool queries
- sync/mod.rs checkout: flat_map+block_on (would panic) -> async for loop

## Key Insights
- In-memory SQLite with sqlx: max_connections(1) is mandatory because each connection creates a separate in-memory DB. This means tests that hold a PoolConnection cannot simultaneously query via &pool -- must drop(c) first or query via &mut *c.
- sqlx::query().execute(conn) moves &mut SqliteConnection in loops. Fix: .execute(&mut *conn) for explicit reborrow.
- async-trait is still needed for dyn GitHubClient dispatch -- native async fn in traits produces unnameable future types that can't go through trait objects.
- daemonize() must run before tokio Runtime::new() -- forking after multi-threaded runtime starts is unsafe.
- set_poll_state: rusqlite allowed ?6 reuse; sqlx anonymous ? doesn't. Fixed with COALESCE in SQL.
