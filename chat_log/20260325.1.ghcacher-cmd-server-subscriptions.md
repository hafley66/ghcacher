# Session: ghcacher-cmd-server-subscriptions

## Goal
Add a subscription/heartbeat system to ghcacher so consumer apps can register interest in repos, keep them cloned and fresh, and optionally drive full PR sync -- all via a lightweight HTTP server.

## Current State
All work committed at 07e1048 on main. 72/72 tests passing.

## Problem/Context
Consumer apps needed a way to:
1. Request a repo be cloned/fetched on demand (`ensure_repo`)
2. Keep subscriptions alive via heartbeat so the daemon knows who is active
3. Optionally opt into full PR/branch API sync per repo via `pr_sync` flag
4. Have subscriptions expire automatically when a consumer dies

## Solution/Approach

### Completed this session

1. **`src/cmd.rs`** -- `Subscriptions` struct (RwLock'd HashMap of uuid -> Sub), TCP HTTP server on `127.0.0.1:7748`, two endpoints:
   - `POST /subscribe {uuid, owner, repo, pr_sync}` -- clone-or-fetch repo, register sub, return `{path}`
   - `POST /heartbeat {uuid}` -- touch last_seen, return `{ok}`
   - Sweep thread prunes expired UUIDs every 10s; `active_pr_sync_repos()` filters inline on `elapsed() < ttl` so eviction is always current

2. **`src/config.rs`** -- added `cmd_port` (default 7748) and `heartbeat_ttl_seconds` (default 30) to `[global]`; `staging_folder` now auto-creates instead of warning

3. **`src/main.rs`** -- `ghcache watch` creates `Arc<Subscriptions>`, clones it for the cmd server thread, passes original to `sync::watch()`; cmd server only spawned if `staging_folder` is set

4. **`src/sync/mod.rs`** -- `run()` gains `extra_repos: &[RepoConfig]` param; `watch()` gains `Option<Arc<Subscriptions>>`, calls `active_pr_sync_repos()` each iteration, builds synthetic `RepoConfig`s (sync_prs=true) for subscribed repos not already in config

5. **`ghcache-client/src/cmd.rs`** -- `ensure_repo(port, uuid, owner, repo, pr_sync) -> Result<PathBuf>` and `heartbeat(port, uuid) -> Result<bool>` over raw TCP HTTP, no new deps

## Key Insights
- TTL enforcement is inline in `active_pr_sync_repos()` via `elapsed() < ttl`, not dependent on sweep timing -- sweep is just memory cleanup
- `ensure_repo` path convention is `{staging}/{owner}/{repo}` (no branch subdir) -- consumer opens with git2 and reads any branch directly
- Subscribed repos with `pr_sync=false` still get cloned/fetched but don't enter the PR sync pipeline
- Repos already in config are filtered out of `extra_repos` to avoid double-syncing
- cmd server is stdlib-only TCP HTTP -- no axum/hyper dep added
